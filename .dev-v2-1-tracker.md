## Phase 2 findings

Cranelift 0.124.3 supports `return_call_indirect` on aarch64:
- `src/isa/aarch64/lower.isle:2596` lowers `return_call_indirect sig_ref ptr args`
- `src/isa/aarch64/inst/emit.rs:3051 Inst::ReturnCallInd` → `emit_return_call_common_sequence` + `IndirectBr { rn }` → `enc_br(rn)` = `BR Xn`
- Direct `Inst::ReturnCall:3036` emits `enc_jump26(0b000101, 0)` = unconditional `B target`

**Constraints (verifier `verifier/mod.rs:1675-1700`)**:
1. callee signature.call_conv must be `CallConv::Tail` (`supports_tail_calls()`)
2. caller's signature.call_conv must MATCH callee
3. result types must match

**Common sequence (emit.rs:3620-3666)**:
- gen_clobber_restore (CallConv::Tail-specific clobber map)
- `ldp fp, lr, [sp], #16` (frame restore)
- maybe SP-adjust (if tail args differ)
- maybe pointer-auth hint
- THEN `BR Xn` (indirect) or `B target` (direct)

So Cranelift's musttail on aarch64 is NOT a single `b` instruction. It is a multi-instruction tail sequence (frame restore + branch). Functionally equivalent to LuaJIT's mcode `b` for the JIT→JIT case, but with frame-restore overhead per transition.

Default = since wasmtime PR #8540 (merged 2024-05-03) the wasmtime engine defaults to `CallConv::Tail` on x86_64/aarch64/riscv64 when tail calls are emitted.

## Phase 3 findings

luna trace JIT current ABI: `TraceFn = unsafe extern "C" fn(*mut i64) -> i64` (jit/trace_types.rs:387). All trace signatures built with `module.make_signature()` → ISA default call_conv (on Apple aarch64 = `CallConv::AppleAarch64`). NOT `CallConv::Tail`.

Current parent→child sequence (trace.rs:2933-2987 `emit_side_trace_or_return`):
- load fn_ptr from `side_trace_cell_addr`
- `call_indirect trace_fn_sig_ref fn_ptr [reg_state]`  ← BLR Xn
- bor sentinel mask: `(1<<63) | ((code & 0x7F) << 56)` into the i64 return
- `return_(&[masked])` ← RET

This is a true round-trip: BLR + child epilogue + RET + ORR + RET. Invariant 4 would replace with `return_call_indirect`.

Workload candidates for trace-JIT-mode dispatcher round-trip:
1. **Sandbox embedders** (`examples/sandbox_demo.rs`): `set_jit_enabled(false)` for instr_budget enforcement. trace_jit defaults ON → trace-JIT-mode.
2. **async_drive embedders** (`vm/async_drive.rs:395`): forcibly sets chunk_jit OFF (RFC §Risks: JIT traces don't honor per-op instr_budget). trace_jit untouched. Cooperative multitasking / Future-based hosts.
